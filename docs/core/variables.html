Variables

The variable is a link, which target can be re-assigned at run time.

The variable declaration syntax is similar to the link's one:

<field> [':'] '=' ['<'] <prefix> <initializer>

where:

  • <field> is a left part of either field or adapter declaration;
  • <initializer> is an initial variable value, which will be assigned to variable at first access, unless a new target assigned first;

  • <prefix> has the following syntax:

    {'``' | '(' '``' <type> ')'}

    and declares a variable interface similarly to the link one; note the two `` signs instead of a single one used in the link declaration.

Variable can be used the same way as links.

Variable Assignment

The variable target can be re-assigned inside an imperative block with an assignment statement:

<target> '=' <value>

where <target> is a variable reference, or just an assignment target, and <value> is a value to assign to the <target>.

Note that unlike initial variable target, the <value> will be resolved and assigned immediately at assignment statement execution.

Also note that the assignment statement syntax is similar to the field override one. That's because the field override is impossible inside imperative blocks.

Variable assignment is also possible inside declarative code. But it requires a special syntax, as otherwise it conflicts with a field override syntax.

The variable assignment syntax acceptable for both imperative and declarative code is based on a local scope declaration syntax:

<target> '$' [<local name>] '=' <value>

where <local name> is an optional name for declared local, which value is the assignment target.

As for any local scope declaration, the local name can be omitted.

Note that such syntax allows to use the declared local inside a value expression. This makes it possible to implement things like increment without re-evaluating of target variable:

1
Array [n] $= $ + 1 ~~ Increment the Nth element of array.

Variable Object

Every variable is an object inherited from Variable and can be used similarly to the link object.
